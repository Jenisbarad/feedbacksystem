================================================================================
STEP-BY-STEP GUIDE: CREATE GITHUB REPOSITORY & DEPLOY TO RENDER
================================================================================

STEP 1: CREATE GITHUB REPOSITORY
================================================================================

1.1. Go to GitHub:
    - Open your web browser
    - Go to: https://github.com
    - Sign in to your account (or create one if you don't have it)

1.2. Create New Repository:
    - Click the "+" icon in the top right corner
    - Select "New repository"

1.3. Repository Settings:
    - Repository name: feedback-form (or any name you like)
    - Description: "Simple feedback form with React and Express"
    - Visibility: Choose Public or Private
    - DO NOT check "Initialize with README"
    - DO NOT add .gitignore or license
    - Click "Create repository"

1.4. Copy Repository URL:
    - After creating, GitHub will show you the repository URL
    - It will look like: https://github.com/YOUR_USERNAME/feedback-form.git
    - Copy this URL (you'll need it in the next step)

================================================================================
STEP 2: PUSH CODE TO GITHUB
================================================================================

2.1. Open Terminal/Command Prompt:
    - Navigate to your project folder: D:\company

2.2. Add Remote Repository:
    Run this command (replace YOUR_USERNAME and REPO_NAME with your actual values):
    
    git remote add origin https://github.com/YOUR_USERNAME/REPO_NAME.git

    Example:
    git remote add origin https://github.com/john/feedback-form.git

2.3. Rename Branch to Main (if needed):
    git branch -M main

2.4. Push Code to GitHub:
    git push -u origin main

    - If prompted, enter your GitHub username and password
    - For password, you may need to use a Personal Access Token instead
    - To create a token: GitHub Settings > Developer settings > Personal access tokens > Generate new token

================================================================================
STEP 3: DEPLOY BACKEND TO RENDER
================================================================================

3.1. Go to Render Dashboard:
    - Visit: https://dashboard.render.com
    - Sign in or create a free account

3.2. Create New Web Service:
    - Click "New +" button
    - Select "Web Service"

3.3. Connect GitHub:
    - Click "Connect GitHub" or "Connect account"
    - Authorize Render to access your repositories
    - Select your repository: feedback-form (or your repo name)

3.4. Configure Backend Service:
    - Name: feedback-form-backend
    - Region: Choose closest to you (e.g., Oregon, Frankfurt)
    - Branch: main
    - Root Directory: server
    - Runtime: Node
    - Build Command: npm install
    - Start Command: npm start
    - Instance Type: Free

3.5. Environment Variables (Optional):
    - Click "Advanced"
    - Add if needed:
      - Key: NODE_ENV
      - Value: production

3.6. Deploy:
    - Click "Create Web Service"
    - Wait 2-5 minutes for deployment
    - Copy the service URL (e.g., https://feedback-form-backend.onrender.com)
    - SAVE THIS URL - you'll need it for frontend!

================================================================================
STEP 4: UPDATE FRONTEND WITH BACKEND URL
================================================================================

4.1. Go back to your project folder

4.2. Update client/src/App.js:
    - The file already uses environment variable: REACT_APP_API_URL
    - We'll set this in Render, so no code change needed
    - But you can update the fallback URL if you want

4.3. Commit any changes:
    git add .
    git commit -m "Ready for deployment"
    git push

================================================================================
STEP 5: DEPLOY FRONTEND TO RENDER
================================================================================

5.1. In Render Dashboard:
    - Click "New +" again
    - Select "Static Site"

5.2. Connect Same Repository:
    - Connect your GitHub repository again
    - Select: feedback-form (or your repo name)

5.3. Configure Frontend Service:
    - Name: feedback-form-frontend
    - Branch: main
    - Root Directory: client
    - Build Command: npm install && npm run build
    - Publish Directory: build

5.4. Environment Variables (IMPORTANT!):
    - Click "Environment"
    - Add environment variable:
      - Key: REACT_APP_API_URL
      - Value: https://YOUR-BACKEND-URL.onrender.com
      (Replace with your actual backend URL from Step 3.6)

5.5. Deploy:
    - Click "Create Static Site"
    - Wait for deployment (2-5 minutes)
    - Your website will be live!

================================================================================
STEP 6: TEST YOUR DEPLOYED WEBSITE
================================================================================

6.1. Visit your frontend URL from Render dashboard

6.2. Fill out the feedback form

6.3. Submit and verify it works

6.4. Check backend logs in Render to see if data is received

================================================================================
TROUBLESHOOTING
================================================================================

Issue: "Repository not found" when pushing
Solution: 
- Check repository name and username are correct
- Make sure repository exists on GitHub
- Verify you have access to the repository

Issue: Authentication failed when pushing
Solution:
- Use Personal Access Token instead of password
- Create token: GitHub Settings > Developer settings > Personal access tokens
- Select scopes: repo (all)
- Use token as password when pushing

Issue: Frontend can't connect to backend
Solution:
- Verify REACT_APP_API_URL environment variable is set correctly
- Check backend URL doesn't have trailing slash
- Ensure backend is deployed and running
- Check CORS is enabled (it should be in our code)

Issue: Build fails on Render
Solution:
- Check build logs in Render dashboard
- Verify all dependencies are in package.json
- Make sure Node.js version is compatible

Issue: Free tier goes to sleep
Solution:
- Free tier services sleep after 15 minutes of inactivity
- First request after sleep takes 30-60 seconds
- This is normal for free tier
- Consider upgrading for always-on service
